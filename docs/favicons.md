# ファビコンの仕組み

記事一覧に表示されるサイトのファビコンはビルド時に自動取得されます。`yarn build` を実行すると以下の流れで処理されます。

1. `src/builder/posts.ts` が各メンバーのRSSフィードから記事を収集し `.contents/posts.json` を生成
2. `src/builder/favicons.ts` が `posts.json` に含まれる全ドメインのファビコンを取得し `public/favicons/` に保存
3. 取得成功したドメインの一覧を `.contents/favicons.json` に書き出し
4. `PostList` コンポーネントが `favicons.json` を参照し、取得済みドメインのみ `<img>` をレンダリング

取得できなかったドメインは `<img>` がレンダリングされず、ホスト名テキストのみ表示されます（404リクエストは発生しません）。

## 新しいメンバーを追加したとき

`members.ts` にメンバーを追加して `yarn build` を実行すれば、そのメンバーの記事ドメインのファビコンが自動的に取得されます。手動での操作は不要です。

## ファビコンが表示されないドメインがある場合

ビルド時のファビコン取得が失敗している可能性があります。以下を確認してください。

1. `public/favicons/` に対象ドメインの `.png` ファイルが存在するか確認する
2. 存在しない場合は以下のコマンドで再取得を試みる

```bash
yarn build:favicons
```

`public/favicons/` はgit管理外（`.gitignore`）のため、ローカルにキャッシュされたファイルがあればスキップされます。再取得したい場合はファイルを削除してから実行してください。
